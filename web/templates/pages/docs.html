{{if eq .Page.Lang "ru"}}Быстрый старт{{else}}Quick start{{end}}

    {{if eq .Page.Lang "ru"}}
  1. Создайте workspace.
  2. Создайте flow в статусе draft.
  3. Добавьте шаг вручную или через Import from cURL.
  4. Проверьте flow через Validate и затем запустите его.
  5. Для локальных сценариев используйте sandbox на http://localhost:8091; если окружение резолвит localhost в [::1] и соединение не поднимается, попробуйте http://127.0.0.1:8091.
    6. {{else}}
  6. Create a workspace.
  7. Create a flow in draft status.
  8. Add a step manually or through Import from cURL.
  9. Validate the flow and then run it.
  10. For local scenarios, use the sandbox service at http://localhost:8091; if your environment resolves it to [::1] and the connection is refused, try http://127.0.0.1:8091.
    11. {{end}}

{{if eq .Page.Lang "ru"}}Импорт шага из cURL{{else}}Import a flow step from cURL{{end}}

{{if eq .Page.Lang "ru"}}Откройте flow, нажмите Import from cURL, вставьте curl-команду и получите предварительно заполненный шаг с методом, URL, заголовками, query, body и retry-параметрами. После этого можно добавить extraction и assertions и сохранить шаг прямо в flow.{{else}}Open a flow, click Import from cURL, paste a curl command, and StageFlow will pre-fill the step with method, URL, headers, query, body, and retry settings. You can then add extraction and assertions before saving the step into the flow.{{end}}

curl -X POST http://sandbox:8091/api/users \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice","email":"alice@example.test"}'

{{if eq .Page.Lang "ru"}}Примеры sandbox{{else}}Sandbox examples{{end}}

  • POST /api/users → POST /api/accounts → POST /api/orders
  • GET /api/unstable {{if eq .Page.Lang "ru"}}для проверки retry{{else}}for retry verification{{end}}
  • GET /api/delay/2500 {{if eq .Page.Lang "ru"}}для timeout-сценариев{{else}}for timeout scenarios{{end}}
  • GET /api/assert/wrong-value {{if eq .Page.Lang "ru"}}для намеренного assertion failure{{else}}for deterministic assertion failures{{end}}
  • GET /api/headers {{if eq .Page.Lang "ru"}}для extraction из заголовков{{else}}for response-header extraction{{end}}

{{if eq .Page.Lang "ru"}}Saved request и flow{{else}}Saved requests and flows{{end}}

{{if eq .Page.Lang "ru"}}Saved request удобен для одиночного HTTP-вызова. Flow нужен, когда есть несколько шагов, extraction переменных между шагами, assertions или retry-логика.{{else}}A saved request is useful for a single HTTP call. A flow is the right choice when you need multiple steps, variable extraction across steps, assertions, or retry logic.{{end}}
